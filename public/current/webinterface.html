Freenet Node Configuration
Web interface
  • Enable the web interface (FProxy)?Whether to enable the web interface and related HTTP services. The web interface is the user interface to Freenet which you can access with your browser.
  • Enable SSL?Enable SSL on the web interface. You will need the 'Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy Files' for it to work.
  • Web interface port The TCP port the web interface should listen on
  • IP address to bind to IP address to bind to
  • Web interface themeSelect a theme for the Freenet web interface
  • Override the CSS with a custom one (WARNING!) That setting allows you to override Freenet's CSS with a custom file. WARNING: CSSes can be dangerous and won't be filtered! use at your own risks. We will allow access to the CSS file from /static/override/ (plus the filename within the folder the file is in), so you can refer to images, @include other CSS files etc. If you make a nice theme, consider mailing devl@freenetproject to get it included in the main distribution ;)
  • Send all themes to browser?If set true, all available themes will be loaded by the browser when viewing the FProxy UI pages, and switchable using your browser's page style selection options.
  • Enable Advanced Mode?Enables display of advanced options and information in the web interface by default. This setting should be turned to false in most cases.
  • Enable extended HTTP method handling?Enable this to allow other HTTP methods then GET and POST to be handled. Required for WebDAV and other HTTP extensions.
  • Enable web interface use of Javascript?Whether or not the web interface should use Javascript "helpers". This setting should be turned to false in most cases. Note that freesites may not use javascript even if this is enabled.
  • Enable web interface use of web pushing (experimental)?Whether or not the web interface should use javascript web pushing to update displayed data without reloading. It requires that javascript is turned on. It is experimental and use at your own risk.
  • Have you completed the first-time configuration wizard yet?Have you completed the first-time configuration wizard yet? If not, the web interface will redirect all your requests to it.
  • Disable progress page when loading pages?Set to not show the progress page when loading a page. WARNING: Most browsers will only use a small number of connections to Freenet, therefore turning off the progress page will reduce performance considerably.
  • Show the panic button?Shows a 'panic button' on the queue page that will remove all downloads and uploads, wipe the cache of recently visited freesites, and clear the master keys file.
  • No confirmation on panic button?If true, the panic button will not ask for confirmation but will act immediately.
  • Public gateway mode? (Only applies to allowed but non-full-access connections)Should we enable public gateway mode? For IPs which are allowed to access Freenet, but are not allowed full access, this option disables the download queue and anything else that might conceivably be abused to attack the Freenet node, while still allowing browsing freesites. They will also only see the default bookmarks, not your bookmarks. IP addresses with full access will be allowed to configure the node. You will still need to configure allowed addresses and bind address to get a true public gateway; you should do this after restarting.
  • Enable persistent HTTP connections? (Read detailed description)Don't enable this unless your browser is configured to use lots of connections even if they are persistent.
  • Enable prefetching of inline images?This may help if your browser only uses a small number of connections to talk to Freenet. On the other hand it may not.
  • Enable activelinks?Should the Browse Freenet page show icons for each bookmarked page? The page will load faster if it doesn't show the icons...
  • Maximum size for transparent pass-through in the web interface where we cannot show progress (e.g. images) Maximum size for images etc on the Freenet web interface. If an image, style sheet etc is larger than this threshold, it will fail, and if possible Freenet will show an error. This is to prevent your browser from waiting forever for a page full of enormous images.
  • Maximum size for transparent pass-through in the web interface where we can show a progress bar (e.g. HTML pages) Maximum size of an HTML page or file, where Freenet can show a progress bar. If the file is larger than this it will ask you whether you want to download it in the background, e.g. to your downloads folder.
  • Hostnames or IP addresses that are allowed to connect to the web interface. May be a comma-separated list of single IPs and CIDR masked IPs like 192.168.0.0/24. Note that these can access the hard disk within the limits set by other config options. They can also see your bookmarks.
  • Hosts having a full access to the Freenet web interface (read warning) Hosts granted full access (i.e. change config settings, restart, etc) to your Freenet node. WARNING: Be very careful who you give full web interface access to!
  • Exclude robots via robots.txt?Whether to serve a /robots.txt telling google, spiders, wget, etc to go away
  • Maximum fproxy connections Maximum number of fproxy connections we will serve at once. Each uses a thread.
  • Allow freesites to refresh themselves periodically: Minimum interval in seconds or -1 for disabled. Allow freesites to refresh themselves periodically with HTML meta refresh: Minimum interval in seconds or -1 for disabled.
  • Allow freesites to redirect to other freesites after a delay: Minimum interval in seconds or -1 for disabled. Allow freesites to redirect to other freesites after a delay with HTML meta refresh: Minimum interval in seconds or -1 for disabled.
  • What should fproxy do when a downloaded file matches the key being fetched, but it has already been filtered?What should fproxy do when a downloaded file matches the key being fetched, but it has already been filtered? RE_FILTER = filter it anyway, since the filter may have been improved since the original download, ACCEPT_OLD = accept the old data (maximum performance), RE_FETCH = ignore the old data.
