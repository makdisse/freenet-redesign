Freenet Node Configuration
Auto-update
  • Check for, and download new versionsShould your node automatically check for new versions of Freenet. If yes, new versions will be automatically detected and downloaded, but not necessarily installed. This setting resets itself always back to false unless your Freenet node runs within the wrapper.
  • Automatically install new versionsShould your node automatically update to the newest version of Freenet, without asking?
  • Where should Freenet look for updates? Where should your Freenet node look for updates? This should be a Freenet URI where updates can be downloaded from. The default URI ends in /update/(version number). To download the source code for an update, you can copy the key below, and change "update" to "update-source". Similarly, "fullchangelog" gives you the long, developer-friendly list of changes, and "installer", "wininstaller", "seednodes" and "iptocountryv4" give other files.
  • Where should Freenet look for the revocation key? URI for the revocation key. If this is found Freenet will display its contents and disable the auto-updater.
  • Update the seednodes automatically?Update the seednodes.fref file automatically over Freenet?
  • Download the latest installers over Freenet?Download the latest installers over Freenet? These will be linked from the "Add a Friend" page.
